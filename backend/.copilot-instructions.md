# GitHub Copilot Project Rules

## Mandatory Context
- ALWAYS scan and understand existing project files before generating code
- DO NOT assume file contents
- Follow existing folder structure and naming conventions
- Reuse existing utilities, middleware, and helpers when available
- Do not duplicate logic already implemented elsewhere

## General
- Use Node.js with Express and ES Modules
- Follow MVC structure (routes, controllers, models)
- Prefer async/await
- Do not generate unused code

## Security
- Always use JWT authentication
- Implement RBAC using middleware
- Never expose secrets or passwords
- Hash passwords using bcrypt
- Validate user input

## Code Style
- Use clear variable names
- Keep controllers thin
- Add comments for authentication and RBAC logic
- Handle errors consistently

## Database
- Use MongoDB with Mongoose
- Use schemas with validation
- Do not use deprecated Mongoose methods

## Output Restrictions
- Do not generate frontend code unless explicitly asked
- Do not generate mock data unless requested
- Do not overwrite existing files without instruction
